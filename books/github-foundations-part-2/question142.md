## Q142: バグ報告の管理にディスカッションではなく GitHub Issues を選ぶ理由とは？

### 選択肢

- ディスカッションは、バグ報告の可視化と追跡を改善します。
- Issues は、バグを追跡し解決するための構造化されたフォーマットを提供します。
- Issues はプルリクエストにリンクされると自動的に解決されます。
