## 🌱 自分にとってのゴール地点を決める
自分は「どのぐらい勉強したらよいか」と思ったことはありませんか？
私もそうでした。

ネットを見ると、`過去5年分やったらいい`、`過去問だけで十分`とよく見かけました。
その投稿者の背景や能力が分からないのに、結果だけを鵜呑みにするのは危険だと感じ、
**自分にとってのゴール地点**が必要と考えて行動しました。

1. まず、IPAのサイトや参考書に記載されている試験の合格点を確認します。
基本情報の場合、科目A/Bともに`600点以上`で合格です。

@[card](https://www.ipa.go.jp/shiken/kubun/fe.html)

2. 次に、自分のケアレスミス率を組み込みます。
自分の場合、試験内容を100%理解しても未知問題や緊張で20%ほど間違えてしまいます。

このような計算式が成り立ちます。

$$
本番の正答率 = 過去問の正答率 * 0.8
$$

本番の正答率を`600点以上`にするには、過去問の正答率を`800点以上`にすることを目標にしました。

:::message
**ポイント**
1. 科目A/Bともに`600点以上`
2. 目標の過去問の正答率は、ケアレスミス補正率を考慮して設定する
:::

## 🌱 自分にとってのスタート地点を見つける
ゴール地点だけだと点のままです。もう1点決めて線を引きます。

1. いきなり過去問題を3回分解き、自分の理解度を数値化します。
現時点で **何を理解していて、何が分かっていないか**を確認します。
自分の場合、
スプレッドシートに「正答の結果」と「人に説明できない単語」を一緒に記録しました。
具体的には`2進数`、`ハード系の単語`、`アルゴリズム`などの問題が理解できていないことを確認できました。
前回、情報セキュリティマネジメント試験に合格したので、出題範囲が被っている問題は大丈夫でした。

![ipa-fe-02](/images/articles/ipa-information-technology-engineer-examination/ipa-fe-02.png)
_実際のスプレッドシート_

:::message
**ポイント**
1. 過去問題を1回分だけだと偶然正解するケースもあるため、最低でも3回分は実施したい
2. 過去問題を解く時は、時間制限を設けません。
:::

## 🌱 勉強する量を意図的に減らす
基本情報の出題範囲は思ったより広く、全部勉強するのは非現実的です。
意図的に学習範囲を徐々に狭めていきます。

自分の場合、**2回連続で正解する問題は3回目も正解します。**
※ただし、回答する間隔は1週間以上空けました。

そういう問題は、完全に理解した分野として苦手な分野に注力しました。


:::message
**ポイント**
1. 連続で正解する問題をどんどん増やしていき、練習しなくてもよい分野を増やします。
:::

## 🌱 答えが分からなくても知っている知識で答える
過去問を解いていると参考書に記載がない問題を見かけます。
全ての過去問を暗記するわけにはいきません。
知っている知識で答えるテクニックを身に付けます。

例えば、公式が公開している`ファインチューニング`についてのサンプル問題です。

今読んでいるあなたが`ファインチューニング`の意味が分からないことを前提に解説します。

```md
問１ 大規模言語モデルを用いた自然言語処理において，事前学習済みのモデルに対して行う，
     ファインチューニングに関する記述として，最も適切なものはどれか。

- ア: 強化学習を行い，最適な結果が得られるようにする。
- イ: 事前学習と同じデータを繰り返し用いて学習を行い，モデルの精度を高めるようにする。
- ウ: 大量のテキストデータを用いて学習を行い，モデルの精度を高めるようにする。
- エ: 特定のデータを用いて追加で学習を行い，目的とするタスクに適用できるようにする。
```

@[card](https://www.ipa.go.jp/shiken/mondai-kaiotu/sg_fe/koukai/2025r07.html)

1. 問題文からキーワードを見つけます。
```md
問１ 大規模言語モデルを用いた自然言語処理において，事前学習済みのモデルに対して行う，
ファインチューニングに関する記述として，最も適切なものはどれか。
```
私がキーワードと感じた単語はこちらです。
- `大規模言語モデル`
- `事前学習済み`
- `ファインチューニング`の`チューニング`

2. キーワードから連想して情報を増やす。

| キーワード | 連想情報 | 分かったこと |
| ---- | ---- | ---- |
| 大規模言語モデル | 大規模言語モデルは**AIの分野**で使われる言葉 | `ファインチューニング`はAIに関する単語 |
| 事前学習済み | **学習**についての問題 | `ファインチューニング`はAIの学習に関する単語 |
| チューニング | ギターでチューニングという言葉を知っている<br>日本語だと **「調整する」** 意味？ |　`ファインチューニング`は事前学習済みのAIに対して調整する単語 |

連想して`ファインチューニング`は事前学習済みのAIに対して調整する単語まで導くことができました。

3. 連想した情報と一致する回答を選ぶ
選択肢を見て`ファインチューニング`と一致する回答を選びますが、
どれも似たような表現で選択肢の違いをよく分かりません。

サンプル問題の選択肢を観察します。

```md
- ア: 強化学習を行い，最適な結果が得られるようにする。
- イ: 事前学習と同じデータを繰り返し用いて学習を行い，モデルの精度を高めるようにする。
- ウ: 大量のテキストデータを用いて学習を行い，モデルの精度を高めるようにする。
- エ: 特定のデータを用いて追加で学習を行い，目的とするタスクに適用できるようにする。
```

ちょうど`,`で文章が分割でき、前半と後半に分けられます。　
下記の表のようにまとめました。


| 選択肢 | 前半(AIに加える条件) | 後半(得られる結果) |
| ---- | ---- | ---- |
| ア | 強化学習を行い | 最適な結果が得られるようにする。 |
| イ | 事前学習と同じデータを繰り返し用いて学習を行い | モデルの精度を高めるようにする。 |
| ウ | 大量のテキストデータを用いて学習を行い | モデルの精度を高めるようにする。 |
| エ | 特定のデータを用いて追加で学習を行い | 目的とするタスクに適用できるようにする。 |

後半の内容はどれもAIを良くする内容が書かれているため、差分はなさそうです。
前半の内容を確認すると、太字の部分が明確な差分であることを確認しました。

- **強化学習**を行い
- **事前学習と同じデータを繰り返し**用いて学習を行い
- **大量のテキストデータ**を用いて学習を行い
- **特定のデータ**を用いて追加で学習を行い

2番目の回答が適切な回答になりそうですね。
※正解は、「イ」です。

:::message
**ポイント**
1. 得られる情報から連想して回答する。
:::

## 🌱 科目A: 1問1.5分以内/科目B: 1問5分以内で回答します。
`試験時間`と`問題数`は下記の表のとおりです。

|  | 科目A | 科目B |
| ---- | ---- | ---- |
| 試験時間 | 90分 | 100分 |
| 問題数| 60問 | 20問 |

平均して **科目A: 1問1.5分以内/科目B: 1問5分以内に回答しない** と全問解けません。
過去問を解くときから意識して練習します。

自分の場合は、ポモドーロタイマーを活用して勉強しておりました。
- 科目Aなら1ポモドーロで16~17問を解く
- 科目Bなら1ポモドーロで4~5問を解く

最初は無理でしたが、
何度も練習して用語などの理解が進むと早く問題を解けるようになりました。

特にこの本を約1か月で2周するぐらいやりこみました。おすすめです。
@[card](https://gihyo.jp/book/2024/978-4-297-14271-1)

:::message
**ポイント**
1. 科目A: その単語の`特徴`や`実際どのように使われているのか`まで理解しました。
2. 科目B: 疑似言語の曖昧な理解がなくなるまで繰り返し問題を解きました。
:::

## 🌱 集中力をコントロールする
当日の試験は、`190分`も集中しなければなりません。
当然、そんな長い時間人間は集中できません。

集中力が低下すると、問題文が頭に入らなくなります。
自分なりの集中力をコントロールする術を身に付ける必要があります。

自分の場合、息を止めて無理やり集中していました。
数秒息を止めると苦しくて脳が冴えてきます。

試験当日もこれを何回か繰り返してなんとか集中できました。

:::message
**ポイント**
1. 息を止めて集中力を回復させる
:::
