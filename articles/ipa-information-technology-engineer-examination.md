## 🌱 自分にとってのゴール地点を決める
1. IPAのサイトや参考書に記載されている試験の合格点を確認する
基本情報の場合、科目A/Bともに`600点以上`で合格です。

2. 自分のケアレスミスする補正率を組み込む
自分の場合、試験内容を100%理解しても未知問題や緊張で20%ほど間違えてしまいます。

:::message
**ポイント**
このような計算式が成り立ちます。

$$
本番の正答率 = 過去問の正答率 * 0.8
$$

本番の正答率を`600点以上`にするには、過去問の正答率を`800点以上`を目指す。
:::

## 🌱 自分にとってのスタート地点を見つける
1. いきなり過去問題を3回分解き、自分の理解度を数値化する。

:::message
**ポイント**
現時点で **何を理解していて、何が分かっていないか**を確認します。
:::

自分の場合、
スプレッドシートに「正答の結果」と「人に説明できない単語」を一緒に記録しました。
`2進数`、`ハード系の単語`、`アルゴリズム`などの問題が理解できていない事を確認出来ました。

前回、情報セキュリティマネジメント試験に合格したので、出題範囲が被っている問題は大丈夫でした。

![ipa-fe-02](/images/articles/ipa-information-technology-engineer-examination/ipa-fe-02.png)
_実際のスプレッドシート_

:::message
**ポイント**
1. 過去問題を1回分だけだと偶然正解するケースもあるため、最低でも3回分は実施したい
2. あとはゴール地点に向けてひらすら勉強する
3. 過去問題を解く時は、時間制限を設けない。
:::

## 🌱 勉強する量を意図的に減らす
基本情報の出題範囲は思ったより広く、全部勉強するのは非現実的です。
意図的に学習範囲を徐々に狭めていきます。

自分の場合、**2回連続で正解する問題は3回目も正解します。**
※ただし、回答する間隔は1週間以上空けました。


:::message
**ポイント**
1. 連続で正解する問題をどんどん増やしていき、苦手な問題のみをひたすら復習する
:::


## 🌱 答えが分からなくても知っている知識で答える
過去問を解いていると参考書に記載がない問題を見かけます。
全ての過去問を暗記するわけにはいきません。
知っている知識で答えるテクニックを身に着けます。

例えば、公式が公開している`ファインチューニング`についてのサンプル問題です。

今読んでいるあなたが`ファインチューニング`の意味が分からない。
答えがわからないことを前提に解説します。

```md

- イ: 事前学習と同じデータを繰り返し用いて学習を行い，モデルの精度を高めるようにする。

- エ: 特定のデータを用いて追加で学習を行い，目的とするタスクに適用できるようにする。
```

## 科目B: 1問5分以内で回答できる練習する。

| 項目 | 内容 |
|------|------|
| 何が起きたか | 100分以内に20問を解くが出来なかった |
| 本当の原因 | 5分以内で1問を解ける |
| 知っておくべきこと | 20問の中に回答に**5分以上かかる問題**が存在する<br> |
| 次からどうすれば | 下記手順を参照 |



〔問題〕
```md
次のプログラム中の [a] と [b] に入れる正しい答えの組合せを，解答群の中から選べ。
関数 function1 が受け取る引数と，関数 function2 が受け取る引数とが同じとき，
二つの関数は同じ値を返す。ここで，引数 n と引数 m は正の整数であり，
引数 m は引数 n よりも 10 以上大きい数とする。
```

〔プログラム〕
```py
○ 整数型: function1(整数型: n, 整数型: m)
整数型: count ← 0
整数型: i
for (iをnからmまで1ずつ増やす)
    if ((i mod 4)が0と等しい)
        count ← count ＋ 1
    endif
endfor
return count

○ 整数型: function2(整数型: n, 整数型: m)
整数型: count ← 0
整数型: tempN ← n
整数型: i, j
for ( [a] )
    if ((tempN mod 4)が0と等しい)
        繰返し処理を終了する
    endif
    tempN ← tempN ＋ 1
endfor
for ([b] )
    count ← count ＋ 1
endfor
return count

```
