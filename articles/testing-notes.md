## 🌱 QA
`QA`は `Quality Assurance`（品質保証）= 品質を、運ではなく再現性あるやり方で達成するための活動

## 🌱 テストで確認する基本項目
### バグ
`バグ`とは、意図しない動きのこと
- 表示/UI：レイアウト崩れ、文字化け、ボタンが押せない
- 入力/バリデーション：必須なのに通る、逆に正しいのに弾かれる
- データ/保存：保存されない、別ユーザーのデータに混ざる、消える
- 遷移/状態：戻ると二重処理、ログイン状態が不正、画面が真っ白

### 性能
`性能`とは、できるとして、どれくらいうまくできるか（質や度合いの話）
- 応答時間（速さ）：クリックしてから表示されるまで何秒か
- 処理能力（スループット）：1分に何件処理できるか
- 同時利用耐性（スケーラビリティ）：同時に何人使っても大丈夫か
- 安定性：長時間動かしても落ちない／メモリが増え続けない
- 資源効率：CPUやメモリをムダに食わない

### ユーザー視点
`ユーザー視点`とは、ユーザーとして使いやすいのか？
- データがちゃんと保存できたのかわかりにくい
- ボタンの位置がわかりにくい
- 背景色が文字色が似ていて文字が読みにくい

## 🌱 チェッキングとテストティング
`チェッキング（Checking）`とは、仕様どおりにできているかを確認すること
`テスティング（Testing）`とは、壊れるところ・リスク・未知を探しにいくを調査すること

## 🌱 回帰テスト
`リグレッションテスト`ともいう。不具合修正後に別の個所が壊れていないかを確認するテスト

## 🌱 テストケース
テストする項目のこと
「入力する条件や動作する前提条件」を網羅的に設定することが大切

| 前提条件 | 入力値 | 手順  | 期待結果 |
| ---- | ---- | ---- | ---- |
| 入力項目ABCが空欄であること | 入力項目A=1<br>入力項目B=2<br>入力項目C=3 | 入力項目ABCに入力値を入力する... | ○○と表示される |

## 🌱 テスト観点
どんな方向性、内容でテストを進めるのかをざっくりとした内容のこと

## 🌱 正常系と異常系
`正常系（Happy Path）`とは、仕様が想定する使い方をしたときに、正しく動くことを確認するテストのこと
`異常系（Unhappy Path / Error Case）`とは、仕様が許可しない入力・状態・障害時に、壊れずに適切に失敗することを確認するテストのこと

## 🌱 不具合の情報は明確に伝える
### 要約
`どこでどうしたらどうなった`の形式で伝える
NG: 正しく動きません。よくわからないけど、エラーと表示される


### 環境
下記などのアプリを起動する基盤になるもの情報
- `OS`: Windows OS, mac OS など
- `ブラウザー`: Chrome, edge, Firefox, IE など
- `Version`: Windows 10, Windows 11 など

### 再現手順
1. 他人でも再現できるように記載する
2. 曖昧な表現をしない
3. 1ステップ1行動で端的に記載する

### 現状(実際の結果)
1. どんな状態であるかを言語化する
2. 言語化が難しいならスクリーンショットと矢印で図で補うのもあり

### 期待結果
1. 本当はどう動いてほしいのかを記載する

## 🌱 探索テスト
テストケースを使わないで「目的」に沿って「考えながら」「自由に」行うテストのこと
※経験や知識が必要とされる難しい手法

1. [学習]: どう動くか把握しながら
2. [テスト設計]: どうテストするか考えて
3. [テスト実行]: 実際に試してみる
4. [学習]: 結果を踏まえて次にどうするのか

### テストチャータ
目指す方向性を決めて探索的テストのこと
`チャータ`とは、探索テストの道しるべになるもの（例：仕様書、お題）

- ガイドブックツアー（マニュアルに着目）
- ランドマークツアー（遷移等に着目）
- スーパーモデルツアー（UI等に着目）

### セッションベースト
「テストチャータ」に時間制限を設けるやり方のこと

1. 1セッション（45分から2時間）で探索テストを行う
2. 1セッションが終わったら、15~20分ぐらいで結果を報告する
3. 上記を繰り返す

##  🌱 不具合を見つけるコツ
1. **繰り返す** : 一定の動作を何回も繰り返すことによって不具合を見つける
  「いいね！」とリロードを何回も繰り返して「いいね！」の数が増減していないか
2. **タイミング** : 何かの処理中に追加で処理をする。
  送信ボタンをダブルクリックすると2回送信される
3. **境界値** : 仕様に記載されている境界値の`±1の値`を入力してみる
  パスワードが8文字必須に対して7文字で入力してみる
  整数の最大値, `unix time(32bit 符号付き Unix time)`の最小時間と最大時間 など
